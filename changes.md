- **Analytics layer — Phase 7 (fifth slice): Waterfall** (Analytics.md §6.4) — a spend bridge from the prior period to the current one, by campaign type, tying together the MoM machinery already built in Phase 5:
  - [chartData.ts](file:///d:/Saarthi-CV/frontend/src/lib/analytics/chartData.ts): `buildWaterfall(rows, {dim, measure, currentFilters, previousFilters, topN})` — start total, top-N-by-|delta| per-segment contributions (sorted by magnitude, not raw value — a segment that shrank a lot matters as much as one that grew), an "Other" fold for the rest, and an end total. **The final bar always renders the real current-period total, never the summed running value** — any float drift from adding up deltas can't desync the chart from the actual figure shown everywhere else on the page. **A loud, explicit constraint written into the function's own doc comment**: additive measures only (cost, conversions, impressions, clicks) — bridging a ratio measure (CTR, CPL) this way would silently misattribute the change, since the sum of per-segment ratio values doesn't equal the blended whole (the exact mix/rate problem `RCA_ENGINE_PLAN.md` exists to solve correctly). 8 new checks pass, including that a zero-delta segment is excluded from the bridge entirely and a brand-new segment (present only in the current period) correctly shows its full value as the delta.
